RLM REPL implementation plan

Turn docs/work/001-rlm-setup-repl/research.md into an ambiguity-free two-milestone implementation plan.

Live streaming first Pyodide REPL second FINAL / FINAL_VAR required

The goal is to keep the harness small while moving from direct prompt/tool access to a REPL-mediated RLM loop.

Milestone 1 compatibility is transitional; after Milestone 2, only the final RLM interfaces and tests remain.

Companion file: plan.md

Problem

Current harness

  • generateText waits for turn completion.
  • CLI prints only final text.
  • Model sees raw user input directly.
  • Direct readFile tool is active.

Target harness

  • Live grey trace output for thinking/text/tool activity.
  • Single active repl tool.
  • Python context contains full turn state.
  • User-visible answer comes only from FINAL.

Milestone strategy

1. Streaming refactor

Switch to streamText, normalize stream parts, render trace events live in grey, and preserve current behavior.

Low-risk: no RLM behavior change yet.

2. RLM REPL mode

Replace active tools with Pyodide repl, inject bounded context wrappers, capture FINAL / FINAL_VAR.

RLM semantics isolated after streaming works.

Architecture

Cache Pyodide runtime if useful, but create a fresh logical Python session for every user turn.

Key interfaces

type AgentTraceEvent =
  | { type: "assistant-text-delta"; text: string }
  | { type: "assistant-reasoning-delta"; text: string }
  | { type: "tool-call"; toolName: string; input: unknown }
  | { type: "tool-result"; toolName: string; output: unknown }
  | { type: "tool-error"; toolName: string; error: unknown }
  | ...;

type RlmContext = {
  userInput: string;
  history: readonly AgentConversationEntry[];
};

runAgentTurn(
  history: readonly AgentConversationEntry[],
  userInput: string,
  options: AgentOptions,
): Promise<AgentTurn>;

AgentTurn.text is captured FINAL/FINAL_VAR output only; ordinary assistant text is trace-only in RLM mode.

Milestone 1: streaming refactor

  • Add src/trace.ts for normalized stream events.
  • Replace generateText with streamText.
  • Iterate fullStream and emit onTrace.
  • Add src/trace-renderer.ts for grey ANSI output.
  • Keep readFile active and preserve final text behavior temporarily.

Acceptance: current CLI still works, but trace output appears live. These compatibility tests are replaced by final RLM tests in Milestone 2.

Milestone 1 test plan

Unit tests

  • Normalize text/reasoning deltas.
  • Normalize tool input/call/result/error parts.
  • Format trace events with grey/reset.

Orchestration tests

  • Fake stream emits trace before resolution.
  • Returned text matches accumulated/final text.
  • Returned history is updated.

Milestone 2: RLM REPL mode

  • Add rlm/context.ts for bounded preview generation and LimitedString/LimitedHistory behavior.
  • Add rlm/repl.ts using Pyodide's built-in console/REPL behavior, not a TypeScript reimplementation.
  • Add tools/repl.ts with schema { code: string }.
  • Use only the repl tool in active model calls.
  • Final output comes from FINAL / FINAL_VAR, never ordinary assistant text.

Default maxFinalizationAttempts = 2 means two total attempts: initial + one reminder using the same REPL session.

RLM contracts

Context limits

  • Prompt preview: 500 chars per top-level key.
  • userInput and history content use LimitedString.
  • history uses LimitedHistory.
  • len, bounded slice, preview, and search helpers guide targeted reads.

Tool feedback limits

  • stdout: 4,000 chars.
  • stderr: 4,000 chars; user-code exceptions return here.
  • resultPreview: 4,000 chars from Pyodide REPL display semantics.
  • Captured final values are not internally truncated.

Finalization and REPL semantics

Final output

  • FINAL(str) outputs the string directly.
  • FINAL(nonString) / FINAL_VAR use sorted, 2-space pretty JSON when possible.
  • Non-JSON-compatible values fall back to Python repr.

Execution

  • Use Pyodide PyodideConsole or equivalent.
  • No TypeScript AST parsing or REPL emulation.
  • No concurrency support in v1; sequential turns must be isolated.

Milestone 2 test plan

  • Preview/context tests ensure deterministic truncation, bounded LimitedString display/slice behavior, and no full long input in the prompt.
  • REPL tests cover Pyodide console expression display, stdout, exceptions-as-stderr, FINAL, and FINAL_VAR.
  • Tool tests verify schema and session delegation.
  • Agent tests verify active tools are exactly ["repl"].
  • Finalization tests verify exact total attempt counts, same-session reminders, and clear failure if no final is set.

Invariants

  • ESM TypeScript and Node >= 22 remain unchanged.
  • typecheck, test, and build remain green.
  • Fresh logical Python session per user turn; sequential turns do not leak globals.
  • RLM active tool set contains only repl.
  • User-visible final output comes only from FINAL / FINAL_VAR.

Stop-and-ask boundaries

  • Generic recursive read limits beyond the known { userInput, history } schema.
  • Python-side file helpers or restoring direct file tools.
  • Persistent Python REPL state across turns.
  • Changing finalization syntax.
  • Adding new dependencies.
  • Large CLI UX changes beyond grey trace + normal final output.
